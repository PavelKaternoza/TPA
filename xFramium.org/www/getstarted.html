Get Started

Get start using xFramium today using the following 4 easy steps

  • Get
  • Configure
  • Create
  • Execute

Get the tools

There are various ways to use xFramium. You can work directly with the source code, download the libraries, download the driver project and work remotely with the web start application. Click on the method that works best for you.

xFramium onlineEasy

You can use the xFramium online launcher to execute your tests without having to download and configure anything. The easiest way to get started with xFramium. Once you have your configuration file click here to start

xFramium Driver ProjectMedium

The xFramium Driver project is used to launch xFramium test suites from your IDE of choice. Download the driver project, unzip it and import it into your favorite Java based IDE. Click here to download the xFramium Driver project

xFramium SourceAdvanced

xFramium is an open source project therefore all of the source code is made available. Clone the GitHub repository and run directly from the latest source.

Configure the tools

Once you have the tools they need to be configured. xFramium offers several methods of configuration to suit any environment including single file solutions and scalable multiple file configurations

Quick StartEasy

The configuration generator is a web based tools to generate your configuration file. This is the place to start. Follow an easy to use workflow to quickly connect xFramium to your testing environment

XML based configurationMedium

Follow the xFramium configuration documentation to manually generate your first configuration file using the xFramium XML format

Text based configurationMedium

Follow the xFramium configuration documentation to manually generate your first configuration file using a simple property file format

JavaAdvanced

xFramium can be completely configured without any property files using Java. You can use Java to configure xFramium and write your test cases on top of our rich framework feature set

Create your test case

Now, you are ready to start creating your test script. If you used the Quick Start generator then there will be a basic test created for you already. Otherwise, start defining your page object and test steps.

XML Keyword-drivenEasy

Define your test cases using a simple, elegant alternative to coding. The XML Keyword-driven method allows you to combine our predefined granular keywords to create complex test cases and function

Behavior-driven development (BDD)Easy

xFramium can use your existing BDD scripts as well. By implementing the underlying functionality as XML based functions your BDD scripts can be used as is without writing Java or Regular Expression parsers.

Matrix based scripts (Excel, SQL, CSV)Medium

Users come from many different backgrounds and xFramium understands that. xFramium test cases can be stored in a SQL database, an Excel workbook or in a CSV file or a combination of all of them in the same test case

Keyword hybrid deploymentMedium

You may have pre-existing functionality written in Java or may find that a particular test case is too complex to express using the keyword language. xFramium allows you mix Java functionality in with your keyword driven tests. xFramiums flexibel configuration model means that a single test suite can contain XML keywords, Excel keywords, BDD CSV and Java based functionality.

Java / TestNGAdvanced

Use Java and TestNG to create your test cases while taking advantage of our feature rich framework libraries to ensure the best test creation process

Execute your test suite

xFramium offers many ways of executing your test suite along with integration into popular CI (Continous Integration) tools. Choose a method of execution that fits into your organization

xFramium onlineEasy

You can use the xFramium online launcher to execute your tests without having to download and configure anything. The easiest way to get started with xFramium. Click here to start

xFramium Driver ProjectEasy

If you had downloaded the xFramium Driver project or have access to the source code you can execute your test case directly from there.

Maven command lineMedium

If you have Maven installed you can use the build in Maven goals to execute xFramium scripts using the xFramium Driver project

Jenkins CI ServerMedium

Jenkins CI server is a continuous integration environment for unattended execution

Java / TestNGAdvanced

Use your IDE or the command line to launch your Java project class or TestNG test classes
